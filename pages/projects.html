Automated Patch Backporting

LangGraph Spring Boot (MCP) Python TypeScript

A multi-agent, Large Language Model (LLM) system framework designed to automate the backporting of security patches in enterprise software (e.g., OpenJDK, Kafka). This research addresses the critical Semantic and Structural Divergence problem‚Äîwhere newer patches rely on APIs that do not exist in older legacy branches‚Äîby mimicking a human engineering team's reasoning process.

1. System Architecture & Agentic Workflow

Unlike standard code generators, our system decomposes the task into specialized roles to ensure semantic correctness using a Call Graph approach.

Reasoning Agent

Uses Call Graphs to trace change propagation and plan the adaptation, separating core logic from side effects.

Generation Agent

Implements the plan using a mix of template-based substitution and LLM synthesis for complex logic.

Validation Agent (3-Layer)
  • Rapid Compilation
  • Static Analysis
  • Regression Testing

2. Methodology & Progress

Dataset Curation (JavaBackports)

Mined 11,000+ commits to build a benchmark of 474 verified backport instances across major open-source projects (Elasticsearch, Hadoop, Kafka).

Hybrid Code Location Engine

Developed a repository-agnostic engine combining Git history tracing with TF-IDF & Vector Embeddings to accurate identify patch locations across divergent branches.

Multi-Agent Implementation

Implemented the orchestration layer and reasoning logic for dependency analysis and initial validation.
